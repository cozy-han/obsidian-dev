# 개발환경 설정 가이드

#kac-utm #backend #setup

> 신입 개발자가 로컬에서 프로젝트를 실행하기 위한 단계별 가이드

---

## 사전 요구사항

| 도구 | 버전 | 용도 |
|------|------|------|
| **JDK** | 17 | Java 컴파일 및 실행 |
| **Gradle** | Wrapper 포함 | 빌드 도구 (별도 설치 불필요) |
| **Docker Desktop** | 최신 | MySQL, Redis, RabbitMQ 실행 |
| **IntelliJ IDEA** | 권장 | IDE (Lombok 플러그인 필수) |
| **Git** | 최신 | 소스코드 관리 |

---

## 1단계: 소스코드 클론

```bash
git clone <저장소-URL>
cd kac-utm-be
```

---

## 2단계: Docker 인프라 실행

프로젝트 루트에서 Docker Compose를 실행하여 MySQL, Redis, RabbitMQ를 띄운다.

```bash
docker-compose up -d
```

실행되는 서비스:

| 서비스 | 컨테이너명 | 호스트 포트 | 접속 정보 |
|--------|-----------|-----------|----------|
| **MySQL** | kac-utm-db | 13306 | DB: `kac-utm`, User: `kac` / PW: `palnet!234` |
| **Redis** | kac-utm-redis | 16379 | PW: `palnet!234` |
| **RabbitMQ** | (docker-compose 정의) | 35672(AMQP), 45672(관리콘솔), 31883(MQTT) | guest / guest |

### RabbitMQ 관리 콘솔
- URL: `http://localhost:45672`
- 활성화 플러그인: MQTT, Shovel, Shovel Management

---

## 3단계: 데이터베이스 초기화

MySQL에 접속하여 초기 데이터를 넣어야 한다. SQL 파일은 `_doc/db/` 폴더에 위치한다.

```bash
# MySQL 접속
mysql -h localhost -P 13306 -u kac -p kac-utm
# 비밀번호: palnet!234
```

실행 순서:
1. `_doc/db/ddl.sql` - 테이블 생성
2. `_doc/db/ddl_fk.sql` - 외래키 설정
3. `_doc/db/dml.sql` - 기본 데이터
4. `_doc/db/dml_code.sql` - 공통 코드 데이터
5. `_doc/db/dml_menu.sql` - 메뉴 데이터
6. `_doc/db/dml_authrt.sql` - 권한 데이터
7. `_doc/db/dml_fwk.sql` - 프레임워크 데이터
8. `_doc/db/gsl/*.sql` - 공간정보 데이터 (행정구역, 공역 등)

---

## 4단계: IDE 설정

### IntelliJ IDEA
1. **프로젝트 열기**: `kac-utm-be` 폴더를 Open
2. **Lombok 플러그인**: Settings → Plugins → Lombok 설치
3. **Annotation Processing**: Settings → Build → Compiler → Annotation Processors → Enable
4. **JDK 설정**: Project Structure → Project SDK → Java 17
5. **Gradle JVM**: Settings → Build → Gradle → Gradle JVM → Java 17

---

## 5단계: 빌드 및 실행

### 전체 빌드
```bash
./gradlew build
```
> 테스트는 기본적으로 비활성화 상태이므로 빌드 시 테스트는 실행되지 않는다.

### 특정 앱 실행
```bash
# web 서버 실행 (포트 8000)
./gradlew :app:app-web:bootRun

# 관리자 서버 실행 (포트 8100)
./gradlew :app:app-mngr:bootRun

# 엔진 서버 실행 (포트 8200)
./gradlew :app:app-engine:bootRun
```

### 실행 시 활성화되는 프로필
로컬 환경에서는 자동으로 `local` 프로필이 적용된다.

## 6단계: 동작 확인

| 서비스 | 확인 URL | 예상 결과 |
|--------|---------|----------|
| web | `http://localhost:8000` | API 응답 |
| mngr | `http://localhost:8100` | API 응답 |
| engine | `http://localhost:8200` | API 응답 |

### REST API 테스트
- `_doc/http/` 폴더에 IntelliJ HTTP Client 테스트 파일이 있음
- `_doc/http/http-client.env.json` 에서 환경별 변수 설정

---

## 주의사항

1. **포트 충돌**: 각 모듈이 고유 포트를 사용하므로, 동시에 여러 모듈을 실행할 때 포트 충돌 주의
2. **Docker 메모리**: MySQL + Redis + RabbitMQ를 동시에 실행하면 메모리 사용량이 높을 수 있음
3. **공간정보 데이터**: GSL 관련 SQL은 용량이 클 수 있으므로 별도 실행 권장
4. **외부 API 키**: 로컬 환경에서는 일부 외부 API(기상청, VWorld 등)가 Mock으로 동작할 수 있음
