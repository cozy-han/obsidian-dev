# KAC-UTM 백엔드 프로젝트 개요

#kac-utm #backend

## 시스템 목적

KAC-UTM(Korea Airports Corporation - Unmanned Traffic Management)은 **한국공항공사의 드론 교통 관리 시스템**이다. 드론의 비행계획 신청/승인, 실시간 비행 감시, 공역 관리, 항공 기상정보 제공, 안전 관제 등을 종합적으로 처리하는 백엔드 시스템이다.

## 모듈 간 통신 구조

### Feign 클라이언트 (HTTP 동기 호출)

```
Web(8000) ←→ Mngr(8100) ←→ Engine(8200)
                ↕               ↕
           Sol(8500)      USS-Pub(9000)
                ↕
           Batch(8800)
```

### WebSocket (실시간 양방향)

```
Engine(8200) ←→ WS(8300) ←→ 클라이언트
USS-Pub(9000) ←→ USS-WS(9300) ←→ 클라이언트
```

### 메시지 큐 (비동기)

```
USS-Pub(9000) → RabbitMQ → PSTY(8400)
```

### MQTT/TCP (외부 장비)

```
Bridge(8900) → MQTT/TCP → 외부 드론 스캐너/장비
```

## 핵심 비즈니스 흐름

### 비행계획 신청 → 승인 흐름
1. 사용자가 **web** 또는 **uss-pub**에서 비행계획 신청
2. **engine**에서 공역 충돌 검사, 안전성 판단
3. 관리자가 **mngr**에서 비행계획 검토/승인
4. 승인 결과가 **psty**(알림)를 통해 사용자에게 통보

### 실시간 비행 감시 흐름
1. 드론에서 위치 데이터를 **bridge**(MQTT/TCP)로 전송
2. **psty**가 RabbitMQ를 통해 데이터 수신 및 처리
3. **ws**/**uss-ws**를 통해 관제 화면에 실시간 표시
4. **engine**에서 이상 상황 감지 시 알림 발생
