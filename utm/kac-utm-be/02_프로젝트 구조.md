## 멀티모듈 구조

KAC-UTM은 Gradle 멀티모듈 프로젝트이다. 크게 3개의 그룹으로 나뉜다.

```
kac-utm-be/
├── app/          ← 실행 가능한 애플리케이션 (bootJar)
│   ├── web/
│   ├── mngr/
│   ├── sol/
│   ├── uss-pub/
│   ├── uss-ws/
│   ├── ws/
│   ├── engine/
│   ├── batch/
│   ├── bridge/
│   ├── psty/
│   └── test-front/
│
├── module/       ← 도메인/비즈니스 로직 (jar)
│   ├── common/
│   ├── conts/
│   ├── dos/
│   ├── fpl/
│   ├── gsl/
│   ├── aerl/
│   ├── psty/
│   └── stats/
│
├── common/       ← 공통 라이브러리 (jar)
│   ├── app/
│   ├── config/
│   ├── connector/
│   ├── exception/
│   ├── models/
│   ├── redis/
│   ├── security/
│   ├── sys/
│   └── utils/
│
├── _doc/         ← 문서 (SQL, HTTP 테스트, UML, 드론 시뮬레이터)
├── _docker/      ← Docker 볼륨 데이터
├── resources/    ← 공용 리소스 (항공정보 등)
├── build.gradle  ← 루트 빌드 파일
├── settings.gradle ← 모듈 등록
└── docker-compose.yml
```

## 모듈 네이밍 규칙

Gradle 프로젝트 이름은 `:그룹:그룹-모듈명` 형식이다.

| 폴더 경로 | Gradle 프로젝트 이름 | 예시 |
|-----------|---------------------|------|
| `app/web` | `:app:app-web` | `./gradlew :app:app-web:build` |
| `module/fpl` | `:module:module-fpl` | `./gradlew :module:module-fpl:build` |
| `common/redis` | `:common:common-redis` | `./gradlew :common:common-redis:build` |

이 네이밍은 `settings.gradle`에서 동적으로 설정된다:
```groovy
// settings.gradle 핵심 로직
["app", "module", "common"].each { moduleName ->
    moduleDir.eachFile { subProjectDir ->
        def projectName = ":${moduleName}:${moduleName}-${subProjectDir.name}"
        include projectName
        project(projectName).projectDir = subProjectDir
    }
}
```

## 빌드 설정 (build.gradle)

### 루트 빌드 파일 핵심 설정

```groovy
// 모든 프로젝트 공통
allprojects {
    group("com.kac.utm")
    version("0.0.1")
    sourceCompatibility = JavaVersion.VERSION_17
}

// 서브프로젝트 공통
subprojects {
    bootJar.enabled = false  // 기본적으로 bootJar 비활성화
    jar.enabled = true       // 일반 jar는 활성화

    // 모든 테스트 비활성화
    tasks.withType(Test) { enabled = false }
}
```

> **핵심 포인트**: 기본적으로 모든 모듈은 라이브러리(jar)로 빌드된다. 실행 가능한 app 모듈만 자체 build.gradle에서 `bootJar.enabled = true`로 오버라이드한다.

## 의존성 관계

### App → Module/Common 의존성

```
┌──────────────────────────────────────────────┐
│                  APP 모듈                      │
│  (web, mngr, sol, uss-pub, engine, batch...) │
└──────────┬───────────────────┬───────────────┘
           │                   │
           ▼                   ▼
┌──────────────────┐  ┌───────────────────┐
│   MODULE 모듈     │  │   COMMON 모듈      │
│  (fpl, gsl,      │  │  (security, redis, │
│   psty, aerl...) │  │   connector,       │
│                   │  │   exception...)    │
└──────────────────┘  └───────────────────┘
```

### 대표적인 의존성 예시 (app-web)

```groovy
// app/web/build.gradle
dependencies {
    implementation project(":module:module-common")
    implementation project(":module:module-conts")
    implementation project(":common:common-sys")
    implementation project(":common:common-app")
    implementation project(":common:common-connector")
    implementation project(":common:common-config")
    implementation project(":common:common-exception")
    implementation project(":common:common-utils")
    implementation project(":common:common-redis")
    implementation project(":common:common-models")
}
```

## 패키지 구조

### 기본 패키지
모든 모듈의 기본 패키지는 `com.kac.utm.*` 이다.

### App 모듈 내부 구조 (계층형)
```
com.kac.utm.{앱명}/
├── config/         ← 설정 클래스 (JPA, QueryDSL, Feign 등)
├── module/         ← 도메인별 기능 (일부 앱)
│   ├── aerl/       ← 항공정보
│   │   ├── api/    ← Controller
│   │   ├── service/← Service
│   │   └── model/  ← DTO/Model
│   ├── fpl/        ← 비행계획
│   ├── gsl/        ← 공간정보
│   └── ...
└── {AppName}Application.java  ← 메인 클래스
```

### Module 모듈 내부 구조
```
com.kac.utm.{도메인}/
├── jpa/
│   ├── entity/     ← JPA 엔티티
│   ├── dto/        ← JPA DTO (Projection)
│   └── repository/ ← Repository (Base + Impl 패턴)
├── model/          ← 비즈니스 모델/DTO
│   └── constant/   ← 상수/Enum
└── service/        ← 비즈니스 서비스
```

## Repository 패턴

이 프로젝트는 QueryDSL과 함께 커스텀 Repository 패턴을 사용한다.

```
EntityRepository (interface)
  └── extends JpaRepository<Entity, ID>

EntityBaseRepository (interface)
  └── 커스텀 쿼리 메서드 선언

EntityBaseRepositoryImpl (class)
  └── EntityBaseRepository 구현 (QueryDSL 사용)
```

## 빌드 명령어 정리

```bash
# 전체 빌드
./gradlew build

# 특정 모듈 빌드
./gradlew :app:app-web:build
./gradlew :module:module-fpl:build

# 특정 앱 실행
./gradlew :app:app-web:bootRun

# REST Docs 생성
./gradlew :app:app-web:asciidoctor

# clean 후 빌드
./gradlew clean build
```

## _doc 폴더 구조

개발에 유용한 문서와 도구가 `_doc/` 폴더에 있다.

```
_doc/
├── db/           ← SQL 스크립트 (DDL, DML, GSL)
├── drone/
│   ├── mqtt/     ← MQTT 드론 시뮬레이터 (Node.js)
│   └── tcp/      ← TCP 프로토콜 테스트
├── http/         ← IntelliJ HTTP Client 테스트 파일
├── example/      ← MQTT 메시지 예제
├── uml/          ← PlantUML 다이어그램 (도메인별)
└── pmd/          ← 정적 분석 결과
```
