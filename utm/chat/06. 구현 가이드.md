## 주의사항

| 항목 | 설명 |
|------|------|
| **STOMP 인증** | HTTP 필터가 아닌 `ChannelInterceptor`에서 CONNECT 프레임의 JWT 검증 |
| **DM 중복 방지** | 두 유저 간 기존 DM 방 존재 여부 QueryDSL로 확인 후 생성 |
| **메시지 커서** | `MSG_SN`이 삽입 순서를 보장 → `WHERE MSG_SN < cursor ORDER BY MSG_SN DESC` |
| **방 나가기** | DM은 숨김 처리(재DM시 복원), 그룹은 마지막 멤버 퇴장 시 방 소프트 삭제 |
| **멀티 인스턴스** | Phase 1은 단일 인스턴스 (`enableSimpleBroker`), Phase 2에서 Redis pub/sub |

## 검증 방법

1. DDL 실행 후 테이블 생성 확인
2. Gradle 빌드 성공 확인
3. `app/chat` 부트 실행 (포트 8400)
4. REST API 테스트 (Postman/curl)
   - 방 생성 → 메시지 전송 → 히스토리 조회
5. WebSocket 테스트 (STOMP 클라이언트)
   - 접속 → 구독 → 메시지 실시간 수신 확인
6. 유저 검색 → 초대 → 시스템 메시지 + 알림 확인
