## 1. 프로젝트 개요

### 1.1 목적

Upbit API를 활용한 암호화폐 자동매매 시스템을 구축한다.
다양한 트레이딩 전략을 플러그인 방식으로 조합/분리하여 적용할 수 있으며, 모의 매매(Paper Trading)를 통한 전략 검증과 백테스팅을 지원한다.

### 1.2 대상 사용자

- 개인 투자자 (본인)
- 추후 소스코드 오픈을 통한 커뮤니티 사용자

### 1.3 거래 대상

- Upbit KRW 마켓 전체 (BTC + 알트코인)
- 마켓 스캔 후 종목을 선택/추가/삭제하여 관리

## 3. 비기능 요구사항

### NFR-01. 성능

| ID        | 요구사항                            | 기준      |
| --------- | ------------------------------- | ------- |

## 5. 시스템 경계 (Scope)

### 포함 (In Scope)

- Upbit KRW 마켓 자동매매 (매수/매도)
- 전략 관리 (생성, 조합, 적용, 활성화/비활성화)
- 실시간 시세 수신 및 캔들 데이터 수집
- Paper Trading (모의 매매)
- 백테스팅 및 시각화
- 리스크 관리 (손절/익절, 하드 리밋, 포지션 관리)
- Discord 알림
- 계좌/자산 조회
- K8s 기반 배포 및 CI/CD

### 제외 (Out of Scope)

- 입금/출금 기능 (알림으로 대체)
- 타 거래소 연동 (Upbit만 대상)
- 모바일 앱
- 사용자 인증/회원가입 (개인 사용 기반)
- 소셜 트레이딩 / 카피 트레이딩
- 선물/마진 거래

### 추후 도입 예정 (Future)

- 대시보드 UI (Grafana 또는 자체)
- 시스템 헬스체크/메트릭 (Prometheus + Micrometer)
- 시계열 전용 DB (TimescaleDB)
- SOA/MSA 전환

## 7. 용어 정의

| 용어 | 설명 |
|------|------|
| 캔들(Candle) | 일정 시간 동안의 시가/고가/저가/종가/거래량 데이터 |
| 타임프레임(Timeframe) | 캔들의 시간 단위 (1분, 5분, 1시간, 1일 등) |
| 전략(Strategy) | 매수/매도 신호를 생성하는 규칙 집합 |
| 복합 전략(Composite Strategy) | 여러 전략을 AND/OR/가중치로 조합한 전략 |
| Paper Trading | 실제 주문 없이 가상 잔고로 전략을 검증하는 모의 매매 모드 |
| 백테스팅(Backtesting) | 과거 데이터로 전략 성과를 시뮬레이션하는 것 |
| 하드 리밋(Hard Limit) | 최대 손실 도달 시 강제로 매매를 중지하는 안전장치 |
| MDD(Maximum Drawdown) | 최대 낙폭. 고점 대비 최대 하락 비율 |
| GitOps | Git을 단일 진실 소스로 사용하여 인프라/배포를 관리하는 방식 |
