### 추천이 필요한 미결정 사항

#### 1. Kubernetes 도구 → **Rancher Desktop** 추천

> 이번 프로젝트뿐 아니라 향후 프로젝트에도 활용 예정

| 이유         | 설명                                                       |
| ---------- | -------------------------------------------------------- |
| 범용성        | K3s 기반이라 Spring Boot, Node.js, Python 등 어떤 프로젝트든 동일하게 사용 |
| 무료         | Docker Desktop과 달리 기업 규모 무관 완전 무료                        |
| GUI        | K8s 버전 전환, 리소스 관리가 GUI로 편리                               |
| dockerd 호환 | 기존 Docker/Docker Compose 워크플로우 그대로 사용 가능                 |
| CI/CD 연동   | GitHub Actions에서 K3d로 동일 환경 테스트 가능                       |

- 상세 비교: [[Kubernetes 로컬 개발 도구 비교]]

#### 2. CI/CD 도구 → **ArgoCD + GitHub Actions** 추천

> K8s에 설치하여 사용 + GitHub Actions도 활용하고 싶음

| 도구 | 역할 | 비용 |
|------|------|------|
| **GitHub Actions** | 빌드/테스트/이미지 푸시 | 무료 (public repo 무제한, private 2,000분/월) |
| **ArgoCD** | K8s 배포 자동화 (GitOps) | 무료 (K8s에 설치) |

```
코드 Push → GitHub Actions (빌드/테스트/이미지) → Git 매니페스트 업데이트 → ArgoCD (K8s 배포)
```

- GitHub Actions: CI (빌드, 테스트, Docker 이미지 빌드/푸시)
- ArgoCD: CD (Git에 정의된 K8s 매니페스트를 감시하고 자동 배포)
- 소스코드 오픈(public repo)이면 GitHub Actions 무료이므로 금전적 부담 없음

#### 3. API Key 관리 → **Sealed Secrets** 추천

> 소스코드 오픈 + K8s 환경

| 이유        | 설명                                            |
| --------- | --------------------------------------------- |
| Git 커밋 가능 | 암호화된 시크릿을 Git에 안전하게 커밋 → 다른 사용자도 구조 확인 가능     |
| K8s 네이티브  | 클러스터 내 컨트롤러가 자동으로 복호화 → K8s Secret으로 변환       |
| 별도 서버 불필요 | Vault와 달리 별도 인프라 운영 불필요                       |
| Spring 통합 | K8s Secret → 환경변수 → Spring `@Value`로 자연스럽게 주입 |

```
개발자 → kubeseal로 암호화 → SealedSecret (Git에 커밋 가능)
                                    ↓ (K8s 클러스터 내)
                            Sealed Secrets Controller → K8s Secret → Pod 환경변수
```

#### 4. 데이터 백업 → **pg_dump + 로컬 스토리지** 추천

| 방식      | 설명                                |
| ------- | --------------------------------- |
| 일별 백업   | `pg_dump`로 매일 자동 백업 (K8s CronJob) |
| 월별 백업   | 월 단위로 압축 아카이브 보관                  |
| 연별 아카이브 | 연 단위로 압축 아카이브 보관                  |
| 보관 경로   | 호스트 PC의 별도 디스크/폴더에 마운트            |
| 추후 확장   | S3 호환 스토리지(MinIO)로 전환 가능          |

#### 5. 백테스팅 시각화 → **자체 웹 UI (Thymeleaf + Lightweight Charts)** 추천

| 이유          | 설명                                        |
| ----------- | ----------------------------------------- |
| 통합성         | Spring Boot 앱 내에서 바로 제공 가능                |
| 캔들 차트       | TradingView Lightweight Charts (무료, 오픈소스) |
| 수익률 그래프     | Chart.js로 백테스팅 결과 시각화                     |
| Grafana 불필요 | 초기에는 오버스펙, 추후 모니터링 단계에서 도입                |

#### 6. 전체 자산 하드 리밋 → 도입 권장

> 종목별 하드 리밋은 확정. 전체 자산은 미결정

- **권장**: 전체 자산 대비 -15% 도달 시 전체 매매 중지 (기본값, 사용자 정의 가능)
- 종목별 하드 리밋만으로는 여러 종목이 동시에 하락할 때 보호 불가
- 전체 하드 리밋은 **최후의 안전장치** 역할
