---

## 2. 실행 순서

### Step 1: PostgreSQL 시작

```bash
cd docker
docker compose up -d
```

- PostgreSQL 16 (Alpine) 기동
- DB: `bitcoin_trader`, User/Password: `bitcoin/bitcoin`
- Port: `5432`
- Healthcheck 포함 (자동 readiness 확인)

### Step 2: 애플리케이션 실행

```bash
./gradlew :app:bootRun --args='--spring.profiles.active=dev'
```

- 서버 포트: `8000`
- 프로필: `dev` (SQL 로깅 활성화)

### Step 3: 정상 동작 확인

```bash
# Health Check
curl http://localhost:8000/actuator/health

# Swagger UI (브라우저)
open http://localhost:8000/swagger-ui/index.html

# Actuator 메트릭
curl http://localhost:8000/actuator/metrics
```

## 4. 환경 변수

### 필수 (기본값 있음)

| 변수 | 기본값 | 설명 |
|------|--------|------|
| `DB_HOST` | `localhost` | PostgreSQL 호스트 |
| `DB_PORT` | `5432` | PostgreSQL 포트 |
| `DB_NAME` | `bitcoin_trader` | 데이터베이스 이름 |
| `DB_USERNAME` | `bitcoin` | DB 사용자 |
| `DB_PASSWORD` | `bitcoin` | DB 비밀번호 |
| `SERVER_PORT` | `8000` | 서버 포트 |
| `TRADING_MODE` | `PAPER` | 매매 모드 (PAPER/REAL) |

### 선택 (API 키)

| 변수 | 기본값 | 설명 |
|------|--------|------|
| `UPBIT_ACCESS_KEY` | (빈 문자열) | Upbit API Access Key |
| `UPBIT_SECRET_KEY` | (빈 문자열) | Upbit API Secret Key |
| `DISCORD_WEBHOOK_URL` | (빈 문자열) | Discord 알림 Webhook URL |
| `DISCORD_ENABLED` | `true` | Discord 알림 활성화 |

> **API 키 없이도 앱은 정상 시작됩니다.**
> - 공개 API (`/market/all`, 캔들, 시세)는 인증 없이 동작
> - 시작 시 마켓 동기화 (237개 KRW 종목) 자동 실행
> - 인증이 필요한 API (계좌 조회, 주문 실행)만 호출 시점에 실패
> - Discord Webhook URL이 비어있으면 알림이 자동으로 스킵됨

## 5. 데이터베이스 관리

### Flyway 마이그레이션

| 버전 | 내용 |
|------|------|
| V1 | 핵심 테이블 10개 (markets, candles, orders, trades, positions 등) |
| V2 | Outbox 패턴 (`outbox_events` 테이블, `event_publication` 제거) |
| V3 | 알림 이력 (`alert_history` 테이블) |

앱 시작 시 Flyway가 자동으로 마이그레이션을 실행합니다 (`baseline-on-migrate: true`).

### DB 초기화 (전체 리셋)

```bash
cd docker
docker compose down -v    # 볼륨까지 삭제
docker compose up -d      # 깨끗한 DB로 재시작
```

### DB 접속

```bash
docker exec -it bitcoin-trader-db psql -U bitcoin -d bitcoin_trader
```

## 6. API 키 설정 (실제 매매 시)

1. [Upbit 개발자 센터](https://upbit.com/service_center/open_api_guide)에서 API 키 발급
2. 환경 변수 설정:

```bash
export UPBIT_ACCESS_KEY=your-access-key
export UPBIT_SECRET_KEY=your-secret-key
```

또는 IntelliJ Run Configuration에 환경 변수 추가.

> **주의**: Secret Key는 256비트 (32바이트) 이상이어야 합니다.

## 7. 트러블슈팅

### 포트 충돌

```bash
# 8000 포트 사용 중인 프로세스 확인
lsof -i :8000
# 5432 포트 확인
lsof -i :5432
```

### Docker 컨테이너 상태 확인

```bash
docker compose -f docker/docker-compose.yml ps
docker compose -f docker/docker-compose.yml logs postgres
```

### Flyway 마이그레이션 실패

DB를 초기화하고 다시 시작:

```bash
cd docker && docker compose down -v && docker compose up -d
```

### 마켓 동기화 실패

API 서버 문제 시 앱은 정상 시작되며 경고 로그만 출력됩니다:

```
WARN  MarketSyncService : Initial market sync failed — app continues without market data
```

## 8. 종료

```bash
# 앱 종료: Ctrl+C (bootRun 중) 또는
kill $(lsof -ti :8000)

# PostgreSQL 종료
cd docker && docker compose down
# 데이터 보존됨 (볼륨 유지)
```
