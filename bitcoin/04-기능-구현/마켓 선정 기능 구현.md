## 개요

모든 KRW 마켓의 일봉 데이터를 수집하고, N일 평균 거래대금 기준으로 상위 마켓을 자동 선별하여 활성화하는 기능.

### 핵심 변경

- **일봉 수집 대상 확대**: 활성 마켓만 → **전체 KRW 마켓**
- **자동 마켓 선정**: 수집된 캔들의 거래대금으로 상위 마켓 활성화
- **시간봉 수집은 유지**: 활성 마켓만 (기존 동작 그대로)

## 스케줄링 타임라인

```
00:05 KST  CandleCollectionService.collectDailyCandles()      — 전체 마켓 일봉 수집 (~24초)
00:10 KST  MarketSelectionService.scheduledMarketSelection()   — 거래량 기반 선정
매시 05분   CandleCollectionService.collectHourlyCandles()      — 활성 마켓만 시간봉
09:00 KST  MarketSyncService.scheduledSync()                   — 마켓 메타데이터 동기화
```

## 설정

```yaml
market:
  selection:
    enabled: true                    # 활성화 여부
    top-n: 20                        # 상위 N개 선택
    min-trade-price-krw: 1000000000  # 최소 거래대금 (10억원)
    average-days: 7                  # 평균 계산 기간 (N일)
```

환경변수로도 설정 가능:
- `MARKET_SELECTION_ENABLED`
- `MARKET_SELECTION_TOP_N`
- `MARKET_SELECTION_MIN_TRADE_PRICE`
- `MARKET_SELECTION_AVG_DAYS`

## 마켓 선정 로직

```
1. DB에서 최근 N일간 모든 일봉 캔들 조회
2. 마켓별 평균 거래대금(accTradePrice) 계산
3. 최소 거래대금 미만 제외
4. 평균 거래대금 내림차순 정렬 → 상위 N개 선택
5. 선택된 마켓 activate(), 나머지 deactivate()
6. 변경이 있으면 MarketSelectionChangedEvent 발행
```

### 제외 조건

- 평균 거래대금 < `minTradePriceKrw` (기본 10억원)
- 캔들 데이터가 없는 경우 기존 선택 유지

## CandleCollectionService 변경 요약

### 변경 전

```
collectDailyCandles() → collectCandles(DAYS) → findByActive(true)
collectHourlyCandles() → collectCandles(MINUTES_60) → findByActive(true)
backfillCandles() → findByActive(true)
```

### 변경 후

```
collectDailyCandles() → collectAllMarketCandles(DAYS) → findAll()        ← 전체 마켓
collectHourlyCandles() → collectCandles(MINUTES_60) → findByActive(true) ← 활성만 (기존 유지)
backfillCandles() → findByActive(true)                                    ← 기존 유지
backfillAllMarketCandles() → findAll()                                    ← 신규
```

공통 로직은 `collectForMarkets(markets, candleType, count)` private 메서드로 추출.

## 테스트

### MarketSelectionServiceTest (7개)

1. 거래대금 상위 N개 마켓을 활성화한다
2. 최소 거래대금 미만 마켓은 topN 이내여도 제외한다
3. 기존 활성 마켓이 상위에서 빠지면 비활성화한다
4. 변경이 없으면 이벤트를 발행하지 않는다
5. 변경이 있으면 MarketSelectionChangedEvent를 발행한다
6. 캔들 데이터가 없으면 기존 선택을 유지한다
7. enabled=false이면 실행하지 않는다

### CandleCollectionServiceTest 추가 (2개)

1. `collectAllMarketCandles`는 전체 마켓을 대상으로 수집한다
2. `backfillAllMarketCandles`는 전체 마켓에 200개씩 요청한다

## API 제한 사항

- Quotation Rate Limit: 10 req/sec (IP 기반)
- 일봉 API: 마켓당 1회 호출, 최대 200개 캔들
- 237개 KRW 마켓 × 1 호출 = ~24초 소요 → 충분히 가능
