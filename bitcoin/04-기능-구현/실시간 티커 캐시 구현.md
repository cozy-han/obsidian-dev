# 실시간 티커 캐시 구현

## 배경

- 캔들 수집은 REST API로 DB에 저장하여 전략 분석에 사용
- 실시간 시세(WebSocket ticker)는 거래 실행 시점에 현재가 확인용으로 필요
- 기존에는 `RiskManagementService`가 자체 `ConcurrentHashMap`으로 관리 → 다른 서비스에서 접근 불가

## 설계 결정

### 캐시 대상
| 데이터 | 소스 | 저장소 | 용도 |
|--------|------|--------|------|
| 캔들 (1분봉/일봉) | REST API | PostgreSQL | 전략 분석 |
| 실시간 ticker | WebSocket | **Caffeine 캐시** | 주문 실행, 손절/익절 |

### 메모리 산정
- 활성 마켓 20~50종목 × MarketInfo ~600B = **약 30KB**
- Caffeine max 500개 기준 = **약 300KB**
- 32GB RAM 대비 무시할 수준

### 캐시 설정
- **TTL: 30초** (WebSocket 끊김 시 stale 데이터 방지, 5초는 너무 짧아 정상 수신 중에도 만료 가능)
- **최대 크기: 500개** (KRW 전체 마켓 ~200개 + 여유)

## 아키텍처

```
WebSocket 수신 → TickerUpdatedEvent 발행
  ├→ TickerCacheAdapter (@EventListener): Caffeine 캐시에 저장
  └→ RiskManagementService (@EventListener): 손절/익절 체크 (캐시에서 조회)
```

### Port & Adapter 패턴

```
core 모듈
└─ TickerCachePort (interface)
     ├─ put(marketInfo)
     ├─ get(marketCode): MarketInfo?
     └─ getAll(): Map<MarketCode, MarketInfo>

api-client 모듈
└─ TickerCacheAdapter (구현체)
     - Caffeine 캐시 기반
     - @EventListener로 TickerUpdatedEvent 자동 캐싱
```

## 변경 파일

| 파일 | 변경 |
|------|------|
| `core/.../TickerCachePort.kt` | 신규 — 포트 인터페이스 |
| `api-client/.../TickerCacheAdapter.kt` | 신규 — Caffeine 기반 구현체 |
| `api-client/.../CacheConfig.kt` | ticker TTL 5초 → 30초 |
| `execution/.../RiskManagementService.kt` | ConcurrentHashMap 제거 → TickerCachePort 사용 |
| `app/.../PipelineSimulator.kt` | InMemoryTickerCache 추가 |

## 활용 방법

```kotlin
// 다른 서비스에서 현재가 조회
@Service
class SomeService(private val tickerCache: TickerCachePort) {
    fun getCurrentPrice(marketCode: MarketCode): Money? {
        return tickerCache.get(marketCode)?.tradePrice
    }
}
```
