## 배경

기존 캔들 수집은 최신 1개(스케줄러) 또는 200개(백필)만 가능했다. 특정 기간의 과거 데이터를 수집하거나, 중단 후 이어서 수집하거나, 누락 데이터를 채우는 기능이 없었다.

### 기존 문제점

| 문제 | 설명 |
|------|------|
| 과거 데이터 수집 불가 | 날짜 범위를 지정하여 수집하는 기능 없음 |
| 중복 엔드포인트 | daily/hourly/backfill이 별도 엔드포인트로 분리 |
| 백필 한계 | 고정 200개만 수집, 유연성 부족 |
| 파티션 부족 | V1에서 현재월+3개월만 생성, 과거 데이터 INSERT 시 에러 |
| API 로깅 없음 | Upbit API 요청/응답 전문 확인 불가 |

## 변경 사항

### 1. Service 메서드 통합

**Before** (6개):
```
collectDailyCandles()
collectHourlyCandles()
collectCandles(type)
collectAllMarketCandles(type)
backfillCandles(type)
backfillAllMarketCandles(type)
```

**After** (3개):

| 메서드 | 마켓 범위 | 용도 |
|--------|----------|------|
| `collectLatestCandles(type, allMarkets)` | allMarkets 파라미터로 제어 | 스케줄러/수동 최신 캔들 수집 |
| `collectCandlesInRange(type, from, to)` | 전체 마켓 | 날짜 범위 순방향 수집 |
| `fillMissingCandles(type, from, to)` | 전체 마켓 | 누락 캔들 감지 및 채우기 |

### 2. Controller 엔드포인트 통합

**Before** (4개):
```
POST /api/candles/collect/daily
POST /api/candles/collect/hourly
POST /api/candles/backfill
```

**After** (3개):

| 메서드 | 경로 | 파라미터 | 설명 |
|--------|------|---------|------|
| POST | `/api/candles/collect` | `type`, `allMarkets` | 최신 캔들 수집 |
| POST | `/api/candles/collect/range` | `type`, `from`, `to` | 날짜 범위 순방향 수집 |
| POST | `/api/candles/collect/fill-gaps` | `type`, `from`, `to` | 누락 캔들 채우기 |

### 3. ExchangeClient `to` 파라미터 추가

```kotlin
// Before
fun getCandles(marketCode, candleType, count): List<Candle>

// After
fun getCandles(marketCode, candleType, count, to: LocalDateTime? = null): List<Candle>
```

- Upbit API의 `to` 파라미터 활용 (해당 시각 이전 캔들 반환)
- UpbitClient에서 `UriBuilder`로 전환, `to`가 non-null이면 쿼리 파라미터 추가

### 4. 순방향 수집 (from → to)

기존 역방향(to→from) 수집은 중단 시 과거 데이터에 빈 구간이 생길 수 있었다. `CandleType.minuteInterval`을 활용한 윈도우 슬라이딩 방식으로 순방향 수집을 구현했다.

#### 알고리즘: `collectForMarketForward`

```
from                                                to
 |--[window 1]--|--[window 2]--|--[window 3]--|      |
```

1. **윈도우 계산**: `cursor + (minuteInterval * PAGE_SIZE)`로 윈도우 끝점 산출
2. **API 호출**: `to=windowEnd`로 요청, 해당 윈도우의 캔들 반환
3. **범위 필터**: `[from, to]` 범위 외 캔들 제거
4. **중복 제거**: DB에 이미 있는 캔들 스킵
5. **커서 전진**: 응답의 가장 최신 캔들 + 1 interval을 다음 커서로 설정

#### 신규 상장 마켓 처리

빈 응답이 와도 중단하지 않고 다음 윈도우로 건너뛴다:

```kotlin
if (candles.isEmpty()) {
    cursor = windowEnd
    continue  // break 대신 continue
}
```

PAGE_SIZE 미만 응답도 동일하게 다음 윈도우로 이동:

```kotlin
if (candles.size < PAGE_SIZE) {
    cursor = windowEnd  // break 대신 윈도우 전진
}
```

### 5. 누락 캔들 감지 및 채우기

`fillMissingCandles`는 DB의 기존 캔들 타임스탬프를 분석하여 갭을 찾고, 해당 구간만 API로 채운다.

#### 갭 감지 알고리즘: `findGaps`

기존 캔들을 시간순 정렬 후 연속 쌍을 비교:

```
기존 캔들: [01-01] [01-02] [01-03] --- --- [01-06] [01-07]
                                    ↑ gap ↑
감지 결과: gap = (01-04, 01-06)
```

- 첫 캔들 이전 갭, 연속 캔들 간 갭, 마지막 캔들 이후 갭 모두 감지
- O(n log n) 정렬 기반, 대량 범위에도 효율적

### 6. API RQ/RS 로깅

`RestClientConfig`에 인터셉터를 추가하여 요청/응답 전문을 로깅한다.

```
>>> GET https://api.upbit.com/v1/candles/days?market=KRW-BTC&count=200 body=(empty)
<<< GET https://api.upbit.com/v1/candles/days?market=KRW-BTC&count=200 status=200 body=[{"market":"KRW-BTC",...}]
```

- 로거: `com.autocoin.apiclient.http` (DEBUG 레벨)
- 응답 본문 1000자 초과 시 truncate
- `BufferingClientHttpRequestFactory`로 응답 본문 재사용 가능하게 처리

### 7. 캔들 파티션 자동 관리

#### V5 마이그레이션

`create_candle_partitions(from, to)` DB 함수를 생성하고, 마이그레이션 시 2017-10(Upbit 서비스 시작) ~ 현재+12개월 파티션을 일괄 생성한다.

```sql
SELECT create_candle_partitions();  -- 기본값으로 전체 생성
SELECT create_candle_partitions('2025-01-01', '2026-12-01');  -- 특정 범위
```

#### 스케줄러

`CandlePartitionService`가 매월 1일 01:00(KST)에 `create_candle_partitions()`를 호출하여 향후 12개월 파티션이 항상 준비된 상태를 유지한다.

## 스케줄러 현황

| 태스크 | 주기 | 설명 |
|--------|------|------|
| `collect-daily-candles` | 매일 00:05 | 전체 마켓 일봉 수집 |
| `collect-hourly-candles` | 매시간 :05 | 활성 마켓 시간봉 수집 |
| `sync-markets` | 매일 09:00 | 마켓 목록 동기화 |
| `select-markets` | 매일 00:10 | 거래대금 기반 마켓 선정 |
| `process-outbox-events` | 60초 간격 | Outbox 이벤트 처리 |
| `retry-failed-outbox-events` | 5분 간격 | 실패 이벤트 재시도 |
| `ensure-candle-partitions` | 매월 1일 01:00 | 캔들 파티션 자동 생성 |

## 사용 예시

```bash
# 최신 일봉 수집 (전체 마켓)
curl -X POST "http://localhost:9000/api/candles/collect?type=DAYS&allMarkets=true"

# 2025년 일봉 순방향 수집
curl -X POST "http://localhost:9000/api/candles/collect/range?type=DAYS&from=2025-01-01T00:00:00&to=2025-12-31T00:00:00"

# 2025년 일봉 누락 데이터 채우기
curl -X POST "http://localhost:9000/api/candles/collect/fill-gaps?type=DAYS&from=2025-01-01T00:00:00&to=2025-12-31T00:00:00"
```
