# 백테스팅 차트 시각화

## 개요

백테스트 결과를 실제 주가 차트(캔들스틱)와 함께 시각화하여 매매 시점과 자산 변화를 직관적으로 확인할 수 있는 기능.

## 주요 기능

### 1. Price + Equity 탭 (동기화)
- **상단**: Lightweight Charts 캔들스틱(OHLC) + 매수(B, 녹색 ▲) / 매도(S, 빨간 ▼) 마커
- **하단**: 자산 곡선 (Line Series)
- 두 차트의 **스크롤/줌/크로스헤어가 동기화**됨

### 2. Overlay 탭 (듀얼 Y축)
- 하나의 차트에 캔들스틱(우측 Y축)과 자산 곡선(좌측 Y축, 파란색) 동시 표시
- 가격 흐름과 자산 변화를 한눈에 비교 가능

### 3. Compare 탭 (전략 비교)
- 여러 전략을 선택하여 동일 기간/종목에 대해 비교 실행
- 상단에 캔들스틱 차트, 하단에 전략별 자산 곡선 색상 오버레이
- 범례에 전략명과 수익률 표시

## 기술 구현

### 백엔드 변경

| 변경 | 설명 |
|------|------|
| `GET /api/candles/stored/range` | 날짜 범위 기반 캔들 조회 엔드포인트 (신규) |
| `BacktestResultEntity.candleType` | candle_type 컬럼 추가 (DB 마이그레이션 V14) |
| `BacktestResultDetailRs.equityCurve` | `emptyList()` 버그 수정 → 거래 내역 balance로 재구성 |
| 모든 DTO에 `candleType` 추가 | 프론트엔드에서 캔들 데이터 조회 시 필요 |

### 프론트엔드

- **Lightweight Charts v4.1.3** (이미 로드되어 있었으나 미사용 → 활용)
  - `addCandlestickSeries()`: OHLC 캔들 렌더링
  - `setMarkers()`: 매매 시점 마커
  - `addLineSeries()`: 자산 곡선
  - `subscribeVisibleTimeRangeChange()`: 시간축 동기화
  - `subscribeCrosshairMove()`: 크로스헤어 동기화
  - `priceScaleId: 'left'`: 듀얼 Y축 (오버레이 모드)
- **Chart.js**: Win/Loss 도넛 차트 유지

### 데이터 흐름

```
백테스트 실행/조회
  ├── POST /api/backtest/run → BacktestResultRs (metrics, trades, equityCurve, candleType)
  └── GET /api/candles/stored/range → CandleRs[] (OHLC 데이터, 별도 fetch)
       └── 프론트엔드에서 합쳐서 차트 렌더링
```

캔들 데이터를 백테스트 응답에 포함하지 않고 **별도 엔드포인트로 분리**한 이유:
- 백테스트 응답 payload 경량화
- 저장된 결과 조회 시에도 동일한 방식으로 캔들 fetch 가능
- 비교 기능에서 동일 캔들 데이터 중복 방지
