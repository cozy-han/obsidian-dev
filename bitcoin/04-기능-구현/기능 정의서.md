## FR-01. 마켓 스캔 및 종목 관리

### FR-01-01. KRW 마켓 전체 종목 스캔 ✅

**구현**: `MarketSyncService.syncMarkets()`
**코드**: `api-client/.../application/MarketSyncService.kt`
**동작**: 앱 시작 시 + 매일 09:00 자동 실행

**확인 방법**:
```bash
# 1. 앱 시작 로그에서 확인
# "Market sync completed: 237 new, 0 updated, 237 total KRW markets"

# 2. DB에서 직접 확인
docker exec -it bitcoin-trader-db psql -U bitcoin -d bitcoin_trader \
  -c "SELECT count(*) FROM markets WHERE market_code LIKE 'KRW-%';"
```

### FR-01-02. 거래 대상 종목 선택 ✅

**구현**: `MarketSelectionService.selectMarkets()`
**코드**: `api-client/.../application/MarketSelectionService.kt`
**동작**: 매일 00:10 KST 자동 실행 (일봉 수집 00:05 이후)

**로직**:
1. 최근 N일(기본 7일) 일봉 캔들에서 마켓별 평균 거래대금 계산
2. 최소 거래대금(기본 10억원) 이상 중 상위 N개(기본 20개) 활성화
3. 나머지 비활성화, 변경 시 `MarketSelectionChangedEvent` 발행

**설정** (`application.yml`):
```yaml
market:
  selection:
    enabled: true
    top-n: 20
    min-trade-price-krw: 1000000000
    average-days: 7
```

**확인 방법**:
```bash
# 로그에서 확인
# "Market selection completed: 20 markets selected out of 237 total"

# DB에서 활성 종목 확인
docker exec -it bitcoin-trader-db psql -U bitcoin -d bitcoin_trader \
  -c "SELECT market_code, korean_name, active FROM markets WHERE active = true ORDER BY market_code;"
```

**관련 문서**: [[마켓 선정 기능 구현]]

### FR-01-03. 거래 대상 종목 추가/삭제 🔄

**상태**: FR-01-02와 동일. 도메인 메서드는 있으나 REST API 없음.

### FR-01-04. 종목별 거래 설정 ✅

**구현**: `MarketTradingSettings` + `TradingModeResolver`
**코드**: `execution/.../domain/MarketTradingSettings.kt`
**동작**: 종목별 모드(LIVE/PAPER), 개별 리스크 설정(투자금, 손절/익절) 지원
**마이그레이션**: V9__market_trading_settings.sql

### FR-01-05. 종목별 현재 상태 조회 🔄

**구현**: `PositionController`
**코드**: `execution/.../api/PositionController.kt`

**확인 방법**:
```bash
# 오픈 포지션 조회 (보유량, 수익률)
curl http://localhost:8000/api/positions

# 매매 상태 (오픈 포지션 수, 중지 여부)
curl http://localhost:8000/api/positions/status
```

**미비점**: 시세 연동 수익률 계산은 WebSocket 구독 필요

## FR-02. 트레이딩 전략

### FR-02-01. 이동평균 크로스오버 전략 ✅

**구현**: `MovingAverageCrossoverStrategy`
**코드**: `strategy/.../domain/impl/MovingAverageCrossoverStrategy.kt`
**파라미터**: 단기 MA(기본 5), 장기 MA(기본 20)

**확인 방법**:
```bash
# 백테스트로 확인 (UI)
open http://localhost:8000/backtest

# 백테스트 API 직접 호출
curl -X POST http://localhost:8000/api/backtest/run \
  -H "Content-Type: application/json" \
  -d '{
    "marketCode": "KRW-BTC",
    "strategyType": "MA_CROSSOVER",
    "candleType": "DAYS",
    "startDate": "2025-01-01T00:00:00",
    "endDate": "2026-01-01T00:00:00",
    "initialBalance": 1000000,
    "shortPeriod": 5,
    "longPeriod": 20
  }'
```

**참고**: 백테스트를 실행하려면 해당 기간의 캔들 데이터가 DB에 있어야 합니다.

### FR-02-02. 전략 플러그인 방식 추가 ✅

**구현**: `TradingStrategy` 인터페이스 + `StrategyFactory`
**코드**:
- 인터페이스: `strategy/.../domain/TradingStrategy.kt`
- 팩토리: `strategy/.../application/StrategyFactory.kt`

**확인 방법**: 새 전략 추가 시 아래 2개만 구현하면 됨:
1. `TradingStrategy` 인터페이스 구현 클래스
2. `StrategyFactory`에 타입 등록

### FR-02-03. 복합 전략 조합 ✅

**구현**: `CompositeStrategy`
**코드**: `strategy/.../domain/impl/CompositeStrategy.kt`

### FR-02-04. AND/OR/가중치 결합 방식 ✅

**구현**: `CombineMode` enum (AND / OR / WEIGHTED)
**코드**: `strategy/.../domain/impl/CompositeStrategy.kt`

**확인 방법**: 단위 테스트 7개 통과
```bash
./gradlew :strategy:test --tests "*.CompositeStrategyTest"
```

### FR-02-05. 전략별 종목/타임프레임 지정 ✅

**구현**: `StrategyConfig` JPA 엔티티 (JSONB 파라미터, marketCode)
**코드**: `strategy/.../domain/StrategyConfig.kt`

### FR-02-06. 전략 활성화/비활성화 ✅

**구현**: `StrategyConfig.active` 필드
**코드**: `strategy/.../domain/StrategyConfig.kt`

**미비점**: REST API 없음 (DB 직접 수정)

### FR-02-07. 추가 전략 (RSI, 볼린저밴드 등) ⏳

**상태**: 미구현. `TradingStrategy` 인터페이스만 구현하면 추가 가능.

## FR-03. 시세 데이터 수집

### FR-03-01. 일봉 캔들 수집 ✅

**구현**: `CandleCollectionService.collectDailyCandles()`
**코드**: `api-client/.../application/CandleCollectionService.kt`
**주기**: 매일 00:05 KST
**범위**: **전체 KRW 마켓** (마켓 선정 분석용 데이터 소스)

**확인 방법**:
```bash
# 로그에서 확인
# "Starting daily candle collection for all markets"
# "Collecting DAYS candles for all 237 markets"
```
```sql
-- 전체 마켓 일봉 데이터 확인
SELECT market_code, candle_type, candle_date_time_kst, trade_price
FROM candles
WHERE candle_type = 'DAYS'
ORDER BY candle_date_time_kst DESC
LIMIT 10;
```

### FR-03-02. 시간봉 캔들 수집 ✅

**구현**: `CandleCollectionService.collectHourlyCandles()`
**주기**: 매시간 :05분

### FR-03-03. 분봉 캔들 수집 ⏳

**상태**: API 호출 인프라는 준비됨 (`CandleType.MINUTES_*`). 스케줄러 미구현.

### FR-03-04. WebSocket 실시간 시세 수신 ✅

**구현**: `UpbitWebSocketClient.subscribeTicker()`
**코드**: `api-client/.../adapter/websocket/UpbitWebSocketClient.kt`

**확인 방법**: 현재 `subscribeTicker()` 수동 호출 필요 (자동 구독 미구현)

### FR-03-05. 시세 데이터 3년 보관 🔄

**구현**: 캔들 테이블 월별 파티셔닝 (V1 마이그레이션)
**미비점**: 파티션 자동 생성 CronJob 미구현 (현재 초기 4개월분만)

### FR-03-06. 시세 데이터 백업 ⏳

**상태**: Phase 7 (배포) 에서 구현 예정

## FR-06. 리스크 관리

### FR-06-01. 1회 최대 투자 금액 ✅

**설정**: `trading.risk.max-investment-per-trade: 100000`
**확인**: 주문 전 `RiskManagementService.checkPreOrderRisk()`에서 검증

### FR-06-02. 손절 기준 ✅

**설정**: `trading.risk.stop-loss-percent: -3.0`
**확인**: `RiskManagementService.onTickerUpdated()`에서 실시간 체크

### FR-06-03. 익절 기준 ✅

**설정**: `trading.risk.take-profit-percent: 5.0`

### FR-06-04. 종목별 하드 리밋 ✅

**구현**: `MarketTradingSettings`에서 종목별 개별 리스크 설정 지원
**코드**: `execution/.../domain/MarketTradingSettings.kt`

### FR-06-05. 전체 자산 하드 리밋 ✅

**설정**: `trading.risk.hard-limit-total-percent: -15.0`
**동작**: 전체 미실현 손실률 ≤ -15% → `tradingHalted = true` (전면 매매 중지)

**확인 방법**:
```bash
# 매매 상태 확인 (tradingHalted 포함)
curl http://localhost:8000/api/positions/status
```

### FR-06-06. 최대 포지션 비율 ✅

**설정**: `trading.risk.max-position-ratio: 50.0`

### FR-06-07. API 호출 제한 준수 ✅

**구현**: Resilience4j RateLimiter (FR-04-06과 동일)

### FR-06-08. 리스크 파라미터 사용자 정의 ✅

**확인**: `application.yml` 또는 환경변수로 모든 파라미터 변경 가능

## FR-08. 알림

### FR-08-01. 매수/매도 체결 알림 ✅

**구현**: `TradingEventListener.onTradeCompleted()`
**코드**: `monitoring/.../listener/TradingEventListener.kt`

**확인 방법**: Discord Webhook 설정 후 매매 체결 시 알림 수신

### FR-08-02. 손절/익절 발동 알림 ✅

**구현**: `RiskEventListener.onRiskBreached()`
**코드**: `monitoring/.../listener/RiskEventListener.kt`

### FR-08-03. 하드 리밋 도달 알림 ✅

**구현**: `RiskEventListener`에서 `HARD_LIMIT_TOTAL` 처리 → CRITICAL 알림

### FR-08-04. 입금/출금 감지 알림 🔄

**구현**: `SystemEventListener`에서 이벤트 수신 → 알림 발송
**미비점**: 이벤트를 자동 발행하는 폴링 서비스 미구현 (수동 이벤트만)

### FR-08-05. 시스템 오류 알림 ✅

**구현**: `SystemEventListener.onSystemError()`
**트리거**: WebSocket 연결 끊김, CircuitBreaker OPEN 등

### FR-08-06. 알림 발송 조건 설정 🔄

**구현**: `discord.enabled` 설정으로 전체 on/off 가능
**미비점**: 이벤트별 세부 on/off 미구현

**알림 이력 확인**:
```bash
# 최근 50건 알림
curl http://localhost:8000/api/alerts
```
