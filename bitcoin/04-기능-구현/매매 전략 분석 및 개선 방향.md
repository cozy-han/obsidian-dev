## 2. 근본 원인 분석

### 문제 1: 단일 지표 의존 (16/19 전략)

**19개 전략 중 16개**가 하나의 기술적 지표만으로 매매 신호를 생성함.

```
RSI가 30 아래 → 위로 크로스 → 무조건 BUY
```

확인 신호가 있는 전략은 **3개뿐**:
- Ichimoku: 가격-구름 관계 + 전환선/기준선 크로스
- ADX: DI 크로스 + ADX >= 25 추세 강도 필터
- Stochastic: %K/%D 크로스 + 과매수/과매도 레벨

### 문제 2: 추세 필터 없음

**어떤 전략도 "현재 추세 방향"을 확인하지 않음.**

| 상황 | 현재 동작 | 올바른 동작 |
|------|----------|-----------|
| 하락 추세에서 RSI 30 반등 | BUY (손실) | HOLD (추세 반대) |
| 상승 추세에서 볼린저 상단 터치 | SELL (수익 놓침) | HOLD (추세 방향) |
| 횡보장에서 MA 크로스 | BUY/SELL 반복 (수수료 소진) | HOLD (추세 없음) |

### 문제 3: 고정 손절/익절

```kotlin
val stopLossThreshold = -3.0%   // 고정
val takeProfitThreshold = 5.0%  // 고정
```

- 암호화폐 일일 변동성: 보통 3~10%
- **-3% 손절이 정상 변동 범위 안에서 발동** → 불필요한 손절 후 반등
- 시장 상황(변동성)에 따라 동적으로 조정되어야 함

### 문제 4: 휩소(Whipsaw)

MA 5/20 크로스 예시:
```
Day 1: 5MA > 20MA → BUY
Day 2: 5MA < 20MA → SELL (손실 + 수수료)
Day 3: 5MA > 20MA → BUY (수수료)
Day 4: 5MA < 20MA → SELL (손실 + 수수료)
→ 4거래 동안 수수료만 0.4% 소진
```

짧은 기간(5일) 이동평균은 노이즈에 극도로 민감하여, 횡보장에서 잦은 거짓 신호 발생.

### 문제 5: 거래 비용 누적

- 왕복 수수료: 0.1% (매수 0.05% + 매도 0.05%)
- 휩소로 20번 거래 시: **2%가 수수료로 소진**
- 단기 전략일수록 수수료 비중 증가

### 문제 6: MFI 전략 confidence 계산 버그

```kotlin
// 현재 (잘못된 계산)
val confidence = (currentMFI - oversold) / (50 - oversold)  // 50이 아닌 다른 값이어야 함

// confidence가 1.0 초과하거나 음수 가능
```

## 3. BacktestEngine 실행 흐름

각 캔들마다 아래 순서로 처리:

```
1. 포지션 보유 중이면 → 손절/익절 체크 (자동 청산)
2. strategy.analyze() 호출 → BUY/SELL/HOLD
3. BUY: 포지션 없음 + 잔고 충분 → 매수
4. SELL: 포지션 있음 → 매도
5. 자산 가치 계산 (현금 + 보유량 × 현재가)
6. MDD 업데이트
```

**주의**: 손절/익절이 전략 분석보다 **먼저** 체크됨. 즉, -3% 도달 시 전략이 HOLD를 원해도 자동 청산됨.

## 4. 개선 방향

### 방향 A: 기존 전략에 필터 추가 (권장, 빠른 효과)

#### A-1. 추세 필터 (Trend Filter)

모든 전략에 공통 적용 가능한 추세 확인 레이어:

```
추세 판단: 50일 EMA 기준
- 가격 > 50 EMA → 상승 추세 → BUY 신호만 허용
- 가격 < 50 EMA → 하락 추세 → SELL 신호만 허용
- 가격 ≈ 50 EMA (±1%) → 횡보 → 신호 무시
```

구현 방식: `TrendFilterDecorator`로 기존 전략을 감싸는 데코레이터 패턴

#### A-2. 거래량 확인 (Volume Confirmation)

```
매수 신호 발생 시:
- 현재 거래량 > 20일 평균 거래량 × 1.5 → 신호 유효
- 거래량 부족 → 신호 무시 (약한 움직임)
```

#### A-3. 연속 신호 필터 (Anti-Whipsaw)

```
같은 방향 신호가 N캔들 연속 발생해야 실행:
- 일봉: 2캔들 연속
- 시간봉: 3캔들 연속
→ 순간적 노이즈 필터링
```

#### A-4. ATR 기반 적응형 손절/익절

```kotlin
// 현재: 고정
stopLoss = -3.0%

// 개선: ATR 기반
val atr14 = calculateATR(candles, 14)
val stopLoss = -(atr14 * 1.5 / currentPrice * 100)  // ATR의 1.5배
val takeProfit = (atr14 * 2.5 / currentPrice * 100)  // ATR의 2.5배
```

변동성이 큰 시장에서는 손절 폭을 넓히고, 안정적인 시장에서는 좁힘.

### 방향 B: CompositeStrategy 활용

이미 구현된 `CompositeStrategy`를 활용하여 전략 조합:

#### 추천 조합 1: ADX + RSI (추세 + 타이밍)
- ADX가 추세 확인 (ADX >= 25)
- RSI가 진입 타이밍 결정
- AND 모드

#### 추천 조합 2: Ichimoku + MACD (장기 추세 + 모멘텀)
- Ichimoku가 큰 방향 결정
- MACD가 진입 모멘텀 확인
- AND 모드

#### 추천 조합 3: SuperTrend + Stochastic (추세 추종 + 과매수/과매도)
- SuperTrend가 추세 방향 제공
- Stochastic이 극단값에서 진입
- WEIGHTED 모드 (SuperTrend 0.6, Stochastic 0.4)

### 방향 C: 구조적 개선 (중장기)

| 개선 | 설명 | 난이도 |
|------|------|--------|
| 다이버전스 탐지 | 가격 vs 지표 괴리 감지 (선행 신호) | 높음 |
| 지지/저항 레벨 | 주요 가격대에서 신호 강화 | 중간 |
| ATR 기반 포지션 사이징 | 변동성에 따라 투자 금액 조절 | 중간 |
| 단계적 진입/청산 | 한 번에 전량이 아닌 분할 매수/매도 | 중간 |
| 시간대 필터 | 특정 시간대에만 거래 (예: 아시아 세션) | 낮음 |

## 5. 우선순위 제안

```
1단계 (즉시): A-1 추세 필터 + A-4 ATR 손절/익절
  → 가장 큰 손실 원인(역추세 매매, 조기 손절) 해결
  → 데코레이터 패턴으로 기존 전략 코드 수정 없이 적용

2단계 (단기): A-2 거래량 확인 + A-3 연속 신호 필터
  → 휩소 감소, 거래 횟수 줄어 수수료 절감

3단계 (중기): B 복합 전략 조합 백테스트
  → 최적 조합 발견 후 Paper Trading 적용

4단계 (장기): C 구조적 개선
  → 다이버전스, 지지/저항, 분할 매매
```

## 7. 참고

- 암호화폐 시장은 전통 주식 시장 대비 변동성이 2~5배 높음
- 단일 지표 전략의 암호화폐 시장 승률: 일반적으로 30~45% (학술 연구 기준)
- 복수 지표 확인 + 추세 필터 적용 시: 50~60%까지 개선 가능
- **높은 승률보다 높은 기대값(Risk-Reward Ratio)이 더 중요**
  - 승률 40%라도 R:R이 1:3이면 수익
  - 승률 60%라도 R:R이 3:1이면 손실
